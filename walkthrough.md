# YouTube Time Estimator 複数インスタンス対応の動作説明

このドキュメントでは、複数の `main.py` （またはコンパイルされた `.exe`）を同時に起動して競合なく動作させるために実装された改修内容と、その動作確認方法について説明します。

## 概要

以前のバージョンでは、複数のアプリケーションを起動すると、同じ `log.txt` や Chromeプロファイルに同時にアクセスしようとしてエラーや競合が発生していました。今回の改修により、各プロセスが完全に独立した環境を持つようになりました。

## 主な改修点と動作の仕組み

### 1. 独立したログファイルの生成
- **変更点**: 共通の `log.txt` を廃止し、開始ボタン押下時に固有のログファイル名（`log_<URL下10桁>_<PID>.txt`）を生成するようにしました。
- **動作**: 複数のインスタンスで別々のURLを入力して「開始」を押すと、それぞれ独立したログファイルに進行状況が書き込まれます。

### 2. Chromeプロファイルの分離
- **変更点**: WebDriverのオプションに `--user-data-dir` を追加し、起動ごとに `chrome_profile_<PID>_<UUID(8桁)>` という一意なディレクトリを作成してプロファイルを分離しました。
- **動作**: セッションやキャッシュ、ポート（`--remote-debugging-port=0`による自動割り当て）の競合が一切起きなくなり、複数ブラウザの同時起動が安定します。
- **後処理**: プロセス停止時、この一時プロファイルは自動的に削除されます。万が一Chromeのプロセス終了が遅れて削除に失敗した場合でも、`ignore_errors=True` を指定しているためアプリはクラッシュせず正常終了します。

### 3. パス解決の堅牢化
- **変更点**: PyInstallerによる `.exe` 化（`--onedir` 等）を前提に、`sys.frozen` の状態を確認する `get_base_dir()` 関数を実装しました。
- **動作**: `.exe` 化された後であっても、AppDataやTempではなく、**必ず `.exe` ファイル本体と同じディレクトリ**に `screenshots` フォルダ、ログファイル、一時プロファイルディレクトリが作成されます。

## 検証と使い方

1. **環境構築とビルド**
   - 必要なパッケージをインストールします：`pip install -r requirements.txt`
   - `build.txt` に記載されているコマンドで `.exe` を作成します：
     ```bash
     pyinstaller --noconfirm --onedir --windowed --icon=icon.ico main.py
     ```
2. **同時実行テスト**
   - ビルドされた `.exe` （または `python main.py`）を**2つ以上同時に起動**します。
   - それぞれのウィンドウに異なるYouTubeのURLを入力し、「開始」ボタンをクリックします。
   - `.exe` と同じディレクトリ内に、別々のログファイル（例: `log_oHg5SJYRHA_12345.txt` と `log_xyz1234567_67890.txt`）が生成され、エラーなく記録されることを確認してください。
   - `screenshots` フォルダに、それぞれの指定タイミング（05秒/55秒）でスクリーンショットが正しく保存されることを確認してください。

> [!NOTE]
> 一時ディレクトリ（`chrome_profile_...`）は実行中のみ存在し、停止すると削除されます。ファイル削除エラーを防ぐため削除前に2秒間の待機（`time.sleep(2)`）を入れています。
