# YouTube投稿時間推定用スクリーンショット取得ツール 実装完了

修正要件に基づいてツールの実装が完了しました。

## 実装された変更内容

- **ディレクトリの作成**: 履歴管理に基づき、`C:\Users\3076m\Desktop\antigravity\YouTubeTimeEstimator\1st` に構築しました。
- **main.py の作成**:
  - `tkinter` によるGUIを実装し、指定された情報をリアルタイムで表示できるようにしました。
  - Seleniumをバックグラウンドスレッドで実行し、UIがフリーズしない設計にしました。
  - **時刻制御ロジックの修正**:
    - `05秒`または`55秒`のターゲット時刻の **3秒前（02秒/52秒）** にページリロードを実行するよう変更しました。
    - リロード完了後、高頻度ループ（5ms間隔）でターゲット時刻を監視し、到達した瞬間にスクリーンショットを取得するようにしました。
    - 何らかの理由でリロードが長引き、ターゲット時刻をすでに過ぎてしまっていた場合は、「missed timing」としてスキップし、次の時間まで待機するリカバリ処理を組み込みました。
- **requirements.txt の作成**: `selenium` および `webdriver-manager` を指定しました。

## 動作確認（ユーザーへのお願い）

以下の手順で動作確認をお願いいたします。

1. **コマンドプロンプト（またはPowerShell）を開く**
2. 作業ディレクトリに移動します：
   ```cmd
   cd C:\Users\3076m\Desktop\antigravity\YouTubeTimeEstimator\1st
   ```
3. 必要なパッケージをインストールします（まだインストールしていない場合）：
   ```cmd
   pip install -r requirements.txt
   ```
4. アプリケーションを実行します：
   ```cmd
   python main.py
   ```
5. 表示されたGUIにYouTubeのURLを入力し、「開始」ボタンを押してください。
6. UIの「次回リロード予定」および「次回撮影予定」が正しく計算され、リロード時刻でページアクセス処理が走り、撮影時刻ジャストでスクリーンショットが取得されることを確認してください。
7. `screenshots` フォルダに生成された画像のファイル名の「秒」が `05` または `55` になっていること、および `log.txt` にズレ（ミリ秒単位）が記録されていることを確認してください。
