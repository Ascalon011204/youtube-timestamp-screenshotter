# YouTube投稿時間推定用スクリーンショット取得ツールの実装計画

指定したURLに対して毎分「05秒」および「55秒」のタイミングでスクリーンショットを自動取得し、YouTubeの「○時間前」表記から正確な投稿時間を推定するための情報を収集するツールを実装します。

## 概要

- **GUIフレームワーク**: `tkinter` を使用し、軽量で依存関係の少ないGUIを構築します。
- **ブラウザ操作**: `selenium` および `webdriver-manager` を用いてGoogle Chromeを制御します。
- **ディレクトリ構成**: 履歴管理を徹底し、`C:\Users\3076m\Desktop\antigravity\YouTubeTimeEstimator\1st` 内に構築します。

## User Review Required

> [!IMPORTANT]
> - **ブラウザの表示について**: 現在ヘッドレスモード（画面非表示）をデフォルトとする予定ですが、動作確認用に画面を表示させた方が良いでしょうか？（コード内で容易に切り替えられるようにフラグを設けます）
> - **撮影時の待機時間**: リロード後、描画の安定化のため「3秒」の待機を設ける予定ですが、短くまたは長くする必要がある場合はお知らせください。

#### [NEW] main.py
アプリケーションのメインコード。以下の機能を実装します：
1. **GUI構築**
   - URL入力用テキストボックス
   - 「開始」ボタン
   - 現在時刻のリアルタイム表示（毎秒更新）
   - 次回撮影予定時刻の表示
   - 実行回数の表示（現在回数 / 120回）
2. **時刻同期ロジック**
   - 現在時刻を取得し、直近の「05秒」または「55秒」を計算します。
   - `time.sleep` による固定待機ではなく、目標時刻と現在時刻の差分から待機時間を算出し、常に時刻同期を行います。
   - 処理は別スレッドで実行し、UIのフリーズを防ぎます。
3. **ブラウザ操作と撮影**
   - Seleniumを使用して指定URLを開きます。
   - 目標時刻（05秒/55秒）に到達した直後にページをリロード（または初回読み込み）します。
   - 描画安定化のために3秒待機後、スクリーンショットを取得します。
   - 画像名は `YYYYMMDDhhmmss_URL下10桁.png` とし、ファイル名に使用できない文字はアンダースコア等に置換します。
   - 計120回（1時間分）の撮影が完了した時点で自動停止します。
4. **ログ出力**
   - `log.txt` を作成し、実行の開始、終了、各回の撮影完了、およびエラー発生時の詳細をタイムスタンプ付きで記録します。

#### [NEW] requirements.txt
必要なパッケージ群（`selenium`, `webdriver-manager`など）を記載します。

## 検証計画

### 手動検証
1. プログラムを起動し、UIが正常に表示されるか確認します。
2. 任意のURL（YouTube動画など）を入力し、「開始」ボタンを押します。
3. UI上の「次回撮影予定時刻」が正しく直近の「05秒」または「55秒」になっているか確認します。
4. 予定時刻に到達した瞬間にコンソール/ログに処理開始が記録され、3秒後に `/screenshots` フォルダに画像が保存されることを確認します。
5. 複数回動作させた際、待機時間がズレることなく常に05秒/55秒を基準に動作し続けるか確認します。
6. UIの実行回数カウントが正常に増加するか確認します。
